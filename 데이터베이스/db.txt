다단계 인덱스 화일에 대한 레코드 연산 방법을 생각해보자. 이 인덱스 사용할 수 있나?
:	데이터필드에 있는 가장 작은 값보다 작은 값이 들어오면? 문제 발생.
	회원의 아이디 파일일 경우, 현재있는 아이디보다 작은 아이디는 삽입되지 않음
	-> 이 인덱스는 사용할 수 없다.
=> N진 탐색트리
포인터를 추가하면 -> 인덱스도 파일이라고 할 수 없으므로 파일이라고 하지 않기로함.

B트리 또는 B+트리
: root -> leaf 까지의 거리가 모두 같다. pull balanced. 항상. 3/2 이상의 노드들이 사용중.

B트리 삽입 알고리즘

B+ 트리
: B트리를 개선. 
: 단말 노드: 값, 포인터 구조. 각 노드들이 연결되어 있다.
: 중복 값이 있음!! 가장 큰 차이.
: 단말노드가 분할하면 단말노드에 그대로 있음. 다른 노드는 안그럼.
: 중간노드와 단말노드의 결합 분할 알고리즘이 다르다.
: B+ 트리의 중간노드 알고리즘 == B트리 알고리즘
: B+ 트리의 단말노드 알고리즘 != B트리 알고리즘
